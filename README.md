# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные

#### Интерфейс IProduct
Содержит данные о товаре.

Поля интерфейса\
`id: string` - уникальный идентификатор\
`description: string` - описание\
`image: string` - адрес изображения\
`title: string` - заголовок\
`category: string` - категория\
`price: number | null` - стоимость

#### Интерфейс IBuyer
Содержит данные о покупателе.

Поля интерфейса\
`payment: TPayment` - способ оплаты\
`email: string` - адрес электронной почты\
`phone: string` - телефон\
`address: string` - адрес доставки

### Модели данных

#### Класс Products
Хранит массив товаров и отдельный товар, выбранный для подробного отображения.

Конструктор:\
Конструктор класса не принимает параметров. Для поля `selectedProduct` устанавливает значение null по умолчанию, для поля `productsArray` - пустой массив.

Поля класса:\
`private productsArray: IProduct[];` - хранит массив всех товаров на странице.\
`private selectedProduct: IProduct | null;` - хранит выбранную карточку товара (или ничего, если карточка не выбрана).

Методы класса:\
`setProducts(products: IProduct[]): void` - заменяет текущий массив новым массивом данных (из входящего параметра) и сохраняет его. Генерирует событие `catalog:change` при любом изменении каталога.\
`getProducts(): IProduct[]` - метод возвращает актуальный массив товаров.\
`getProductById(id: string): IProduct | undefined` - возвращает карточку товара для подробного отображения. Входящий параметр - уникальный идентификатор товара, если он не найден, вернется undefined.\
`setSelectedProduct(product: IProduct): void` - сохраняет выбранный товар (из параметра) для подробного отображения.\
`getSelectedProduct(): IProduct` - метод возвращает выбранную карточку товара.

#### Класс Cart
Хранит массив товаров, выбранных покупателем для покупки.

Конструктор:\
`constructor() {this.purchasesArray = [];}` - входящих параметров нет, по умолчанию инициализируется пустой массив, т.к. корзина пуста.

Поля класса:\
`private purchasesArray: IProduct[];` - хранит массив товаров, выбранных покупателем для заказа.

Методы класса:\
`getCartItems(): IProduct[]` - метод возвращает массив товаров, добавленных в корзину.\
`addToCart(product: IProduct): void` - если цена товара не `null` - добавляет товар (из параметра) в корзину. Генерирует событие `cart:change` при любом изменении корзины.\
`removeFromCart(productId: string): void` - удаляет товар из корзины, используя уникальный идентификатор (передан в параметре). Генерирует событие `cart:change` при любом изменении корзины.\
`clearCart(): void` - метод присваивает полю `purchasesArray` пустой массив. Генерирует событие `cart:change` при любом изменении корзины.\
`getTotalPrice(): number` - метод возвращает общую стоимость всех товаров в корзине.\
`getQuantityItems(): number` - метод возвращает количество товаров, находящихся на данный момент в корзине.\
`isInCart(productId: string): boolean` - метод проверяет по идентификатору наличие в корзине выбранного товара.

#### Класс Customer
Хранит информацию о покупателе.

Конструктор:
```
constructor() {
    this.payment = null;
    this.address = '';
    this.phone = '';
    this.email = '';
}
```
В конструкторе нет входящих параметров, т.к. форма заполняется постепенно. Полям присваиваются пустые значения, т.к. форма по умолчанию не заполнена.

Поля класса:\
`private payment: TPayment | null` - тип оплаты (если не выбран, то null)\
`private address: string` - адрес\
`private phone: string` - номер телефона\
`private email: string` - электронная почта

Методы класса:\
`setPayment(payment: TPayment): void` - устанавливает и сохраняет значение способа оплаты.\
`setAddress(address: string): void` - устанавливает и сохраняет адрес.\
`setPhone(phone: string): void` - устанавливает и сохраняет номер телефона.\
`setEmail(email: string): void` - устанавливает и сохраняет электронную почту.\
`getAllData(): IBuyer` - позволяет получить все данные о покупателе сразу в виде объекта покупателя.\
`clearCustomerInfo():void` - удаляет все данные о покупателе.\
`validateField(): TBuyerErrors` - метод проверки полей на пустоту, возвращает объект типа `TBuyerErrors`, поля которого содержат сообщения об ошибках.

### Слой коммуникации

#### Класс BusinessApi

Осуществляет связь приложения с сервером, используя базовый класс `Api`, но добавляя конкретную бизнес-логику.

Конструктор:\
`constructor(api: IApi)` - в конструктор передается объект, соответствующий интерфейсу `IApi` (а именно - экземпляр класса Api).

Поля класса:\
`private api: IApi;` - хранит объект, соответствующий интерфейсу `IApi` (а именно - экземпляр класса Api).

Методы класса:\
`getAllProducts(): Promise<IAllProductsResult>` - делает get-запрос на заданный эндпоинт и возвращает промис с ответом.\
`postOrder(orderInfo: IOrderRequest): Promise<IOrderResult>` - отправляет с помощью параметра `orderInfo` данные пользователя на сервер (на заданный эндпоинт), возвращает промис с объектом-результатом (подтверждением закза и суммой покупки), соответствующим интерфейсу `IOrderResult`.

### Слой представления

#### Класс Header

Отвечает за блок разметки с классом `header`. Контролирует в шапке сайта кнопку иконки корзины и отображение количества товаров в ней. Является потомком базового класса `Component`. 

Конструктор:\
`constructor(protected events: IEvents, container: HTMLElement)` - в конструктор передается событие, которое будет сгенерировано при клике пользователем на кнопку корзины, а также контейнер, в котором будет происходить поиск элементов. Вызывает конструктор базового класса `Component<IHeaderData>`. Найденные элементы будут присвоены полям класса. 

Поля класса:\
`protected basketButton: HTMLButtonElement` - отвечает за отображение кнопки-иконки корзины.\
`protected counterElement: HTMLElement` - отвечает за отображение количества товаров у кнопки-иконки корзины.

Аксессоры класса:\
`set counter(value: number)` - сеттер для свойства counter, позволяет установить значение счетчика количества товаров у кнопки-иконки корзины.

#### Класс Gallery

Отвечает за отображение списка карточек товаров. Является потомком базового класса `Component`.

Конструктор:\
`constructor(container: HTMLElement)` - в конструктор передан контейнер, в котором будет происходить поиск нужных элементов. Вызывает конструктор базового класса `Component<IGalleryData>`

Поля класса:\
`protected catalogElement: HTMLElement` - хранит HTML-элемент каталога товаров.

Аксессоры класса:\
`set catalog(elements: HTMLElement[])`- заменяет текущее содержимое каталога новым контентом.

#### Класс Modal

Отвечает за управление и контроль над модальным окном. Является потомком базового класса `Component`. Содержимое модального окна динамически меняется, в зависимости от события.

Конструктор:\
`constructor(protected events: IEvents, container: HTMLElement)` - в конструктор передается событие, которое будет сгенерировано при клике пользователем на кнопку закрытия модального окна или за его пределами, а также контейнер, в котором будет происходить поиск элементов. Вызывает конструктор базового класса `Component<IModal>`. Найденные элементы будут присвоены полям класса.

Поля класса:\
`protected modalContainer: HTMLElement` - отвечает за отображение элемента, в который будет помещен динамический контент.\
`protected modalButton: HTMLButtonElement` - отвечает за отображение кнопки закрытия.

Методы класса:\
`open(): void` - открывает модальное окно с помощью добавления класса `modal_active` к HTML-элементу модального окна.\
`close(): void` - закрывает модальное окно с помощью удаления класса `modal_active` у HTML-элемента модального окна.

Аксессоры класса:\
`set content(contentElement: HTMLElement)` - устанавливает новое содержимое модального окна, заменяя старое.

#### Класс Success

Отвечает за отображение сообщения об успешном заказе в модальном окне. Выводит сумму заказа, содержит собственную кнопку закрытия окна. Является потомком базового класса `Component`.

Конструктор:\
`constructor(protected events: IEvents,container: HTMLElement,)` - в конструктор передается событие, которое будет сгенерировано при клике пользователем на собственную кнопку закрытия, а также контейнер, в котором будет происходить поиск элементов. Вызывает конструктор базового класса `Component<ISuccess>`. Найденные элементы будут присвоены полям класса.

Поля класса:\
`protected successDescriptionElement: HTMLElement` - отвечает за отображение текста с итоговой суммой заказа.\
`protected successButtonClose: HTMLButtonElement` - отвечает за отображение собственной кнопки закрытия.

Аксессоры класса:\
`set totalPrice(value: number)` - устанавливает итоговую стоимость покупки в сообщении об успешном заказе.

#### Класс Card

Родительский абстрактный класс-дженерик для всех карточек товаров. Содержит в себе общий функционал для дочерних классов. Является потомком базового класса `Component`.

Конструктор:\
`constructor(container: HTMLElement)` - в конструктор передан контейнер, в котором будет происходить поиск нужных элементов. Вызывает конструктор базового класса `Component<T extends TCard = TCard>`. Найденные элементы будут присвоены полям класса. Если опциональные поля будут найдены в контейнере, они тоже будут инициализированны.

Поля класса:\
`protected titleElement: HTMLElement` - хранит элемент, отображающий название товара.\
`protected priceElement: HTMLElement` - хранит элемент, отображающий цену товара.\
`protected imageElement?: HTMLImageElement` - опциональное поле, хранит элемент изображения.\
`protected categoryElement?: HTMLElement` - опциональное поле, хранит элемент отображения категории.

Аксессоры класса:\
`set title(value: string)` - устанавливает название товара.\
`set price(value: number | null)` - устанавливает цену товара.\
`set image(value: string)` - если поле `imageElement` существует - сеттер устанавливает изображение товара, используя метод `protected setImage(element: HTMLImageElement, src: string, alt?: string)`, унаследованный от базового класса `Component`.\
`set category(value: string)` - если поле `categoryElement` существует - сеттер присваивает категорию товару и меняет цвет, если переданная категория соответствует ключу из `categoryMap`.

#### Класс CardCatalog

Потомок класса `Card`, наследует его поля и сеттеры. Отвечает за отображение карточки товара в каталоге. 

Конструктор:\ 
`constructor(container: HTMLElement, actions?: ICardActions)` - в конструктор передан контейнер, в котором будет происходить поиск нужных элементов, а также событие (виде колбэка), которое будет сгенерировано при клике на карточку. Вызывает конструктор базового класса `Card<TCardCatalog>`. Автоватически инициализирует `imageElement` и `categoryElement`.

Поля класса:\
Класс наследует поля от `Card`. Своих полей не добавляет.

Аксессоры класса:\
Наследует все сеттеры от `Card`.

#### Класс CardBasket

Потомок класса `Card`, наследует его поля и сеттеры. Отвечает за отображение карточки товара в корзине.

Конструктор:\
`constructor(container: HTMLElement, actions?: IBasketActions)` - в конструктор передан контейнер, в котором будет происходить поиск нужных элементов, а также событие (виде колбэка), которое будет сгенерировано при клике на кнопку удаления товара из корзины. Вызывает конструктор базового класса `Card<TCardBasket>`.

Поля класса:\
`protected numberInBasket: HTMLElement` - хранит элемент порядкового номера товара в корзине.\
`protected buttonRemove: HTMLButtonElement` - хранит эллемент, отображающий кнопку удаления товара из корзины.

Аксессоры класса:\
`set index(value: number)` - устанавливает порядковый номер товара в корзине.

#### Класс CardPreview

Потомок класса `Card`, наследует его поля и сеттеры. Выводит подробную информацию о товаре в модальное окно.

Конструктор:\
`constructor(container: HTMLElement)` - в конструктор передан контейнер, в котором будет происходить поиск нужных элементов. Вызывает конструктор базового класса `Card<TCardPreview>`. Найденные элементы будут присвоены полям класса.

Поля класса:\
`protected descriptionElement: HTMLElement` - хранит элемент, отображающий описание товара.\
`protected buyButtonElement: HTMLButtonElement` - хранит элемент, отображающий кнопку покупки (добавления товара в корзину).\
`private currentHandler?: () => void` - опциональное поле, хранит ссылку на текущую функцию-обработчик. Нужно для коррекной обработки кнопки, содержимое которой меняется динамически.

Аксессоры класса:\
`set description(value: string)` - устанавливает текст описания. Остальные сеттеры наследует от `Card`.

Метод класса:\
`setButton(text: string, disabled: boolean, onClick?: () => void)` - нужен для настройки и изменения содержимого кнопки. В параметрах - текст кнопки, ее активация/деактивация, опциональная функция, которая будет вызвана при клике на эту кнопку. Все предыдущие обработчики удаляются.

#### Класс Basket

Конструктор:\

#### Класс Form

Абстрактный класс-дженерик. Потомок класса `Component`. Является родительским классом для форм. Содержит в себе общий функционал для дочерних классов.

Конструктор:\
`constructor(container: HTMLElement, protected events: IEvents,)` - в конструктор передается событие, которое будет сгенерировано при клике пользователем на кнопку или при изменении данных в поле ввода, а также контейнер, в котором будет происходить поиск элементов. Вызывает конструктор базового класса `Component<T extends object = object>`. Найденные элементы будут присвоены полям класса.

Поля класса:\
`protected formElement: HTMLFormElement` - хранит элемент формы.\
`protected buttonSubmit: HTMLButtonElement` - элемент, отображающий кнопку формы.\
`protected errorElement: HTMLElement` - хранит элемент с текстом ошибки.\
`protected inputElements: HTMLInputElement[]` - массив полей ввода, найденных в контейнере. 

Метод класса:\
`protected takeData(): Partial<T>` - отвечает за сбор данных со всех найденных полей ввода. Возвращает объект с типом `Partial<T>`, чтобы классы-наследники могли расширять сбор данных в зависимости от условий.

Аксессоры класса:\
`set err(value: string)` - отвечает за вывод текста ошибки.\
`set enableButton(value: boolean)` - отвечает за блокировку кнопки отправки.

#### Класс FormOrder

Отвечает за отображение формы заказа на первом шаге - выбор способа оплаты и адреса доставки. Потомок класса `Form`.

Конструктор:\
`constructor(container: HTMLElement, events: IEvents)` - вызывает конструктор базового класса `Form<TFormOrder>`. В конструктор передается событие, которое будет сгенерировано при клике пользователем на одну из кнопок выбора способа оплаты, а также контейнер, в котором будет происходить поиск элементов. Найденные элементы будут присвоены полям класса.

Поля класса:\
`protected buttonPayOnline: HTMLButtonElement` - отвечает за отображение кнопки, отвечающей за оплату картой онлайн.\
`protected buttonPayCash: HTMLButtonElement` - отвечает за отображение кнопки, отвечающей за оплату наличными.

Метод класса:\
`private selectPay(pay: "card" | "cash")` - устанавливает один из двух возможных видов оплаты, генерирует событие на основе этого выбора.\
`protected takeData(): Partial<TFormOrder>` - переопределен, собирает данные поля ввода (как это реализовано в классе-родителе), но добавляет в объект с данными новое поле с оплатой, определяет активную кнопку по модификатору.

#### Класс FormContacts

Отвечает за отображение формы ввода контактных данных на втором шаге. Использует функционал класса `Form`. Своих полей, методов и аксессоров не требует.

### События

События слоя представления:\
`basket:open` в классе `Header`. Генерируется при клике по кнопке-иконке корзины в шапке сайта. Открывает модальное окно с корзиной.\
`modal:close` в классах `Modal` и `Success`. Закрывает модальное окно при клике на кнопку закрытия, либо за пределами модального окна, либо по кнопке в окне подтверждения заказа.\
`basket:order` в классе `Basket`. Генерируется при клике по кнопке оформления заказа в корзине.\
`order:change` в классе `FormOrder`. Генерируется при изменении любого поля в форме заказа (способ оплаты и адрес).\
`order:submit` в классе `FormOrder`. Генерируется при отправке формы со способом оплаты и адресом при нажатии на кнопку. Переводит на вторую форму с контактами. \
`contacts:change` в классе `FormContacts`. Генерируется при изменении любого поля в форме контактов.\
`contacts:submit` в классе `FormContacts`. Генерируется при отправке формы контактов при нажатии на кнопку. Отправляет собранные данные на сервер.\
События презентера:\
`card:select` в классе `СardCatalog`. Передано через колбэк. При клике по карточке товара в каталоге открывает модальное окно с подробным отображением карточки.\
События слоя моделей данных:\
`catalog:change` в классе `Products`. Генерируется при изменении массива товаров в каталоге.\
`cart:change` в классе `Cart`. Генерируется при любом изменении корзины - полной очистке, добавлении или удалении товара.

### Презентер

Код презентера представлен в основном файле main.ts. Презентер связывает компоненты моделей данных с компонентами представлений с помощью брокера событий `EventEmitter`. Обрабатывает события моделей и представлений, обновляя модели и перерисовывая представления.\
`larekApi.getAllProducts()` - получает с сервера данные с товарами и сохраняет их в модели. На изменение реагирует `catalog:change`, выводит данные в виде карточек.\
При клике на карточку, товар отобразится в модальном окне, в котором будет динамическая кнопка "Добавить в корзину". При клике на нее генрируется событие `cart:change`, изменения отправятся в модели, через презентер будет перерисована сама кнопка - "Удалить из корзины". Добавить товар в корзину можно только один раз.\
При нажатии на кнопку-иконку в шапке сайта откроется модальное окно с корзиной, в которой находится список товаров. Счетчик на иконке меняется в соответствии с количеством товаров в корзине. Каждый товар из корзины можно удалить.\
Если корзина не пуста - будет активна кнопка оформления заказа, которая переводит на форму с оплатой. При некорректном заполнении полей формы появляется сообщение об ошибке, при корректном - активируется кнопка, которая переводит на форму контактов, кнопка которой сохраняет все собранные данные и отправляет их на сервер. После чего появляется модальное окно с сообщением об успешном заказе и его стоимости.\
Модальное окно закрывается при клике на специальную кнопку, при клике за пределами модального окна, при клике на кнопку "За новыми покупками" при успешном заказе, при клике на кнопку "Удалить из корзины" в подробном отображении карточки товара.

Вспомогательная функция:\
`function updateBasket(): void` - обновляет и перерисовывает список товаров в корзине.
